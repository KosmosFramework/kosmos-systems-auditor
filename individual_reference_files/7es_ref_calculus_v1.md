### 5.2 Distinguishing Genuine vs. Artificial Subsystem Boundaries

**Genuine Subsystems:**
- Utilize fundamentally different mechanisms (mechanoreceptors vs. chemoreceptors, flavor vs. mass eigenstates)
- Respond to qualitatively different stimuli (visible light vs. infrared, charged-current vs. neutral-current interactions)
- Project to partially or fully segregated processing centers
- Can be selectively lesioned/disabled without eliminating entire element function
- Evolved or were designed independently for different purposes

**Artificial Boundaries (single subsystem with internal variation):**
- Variants of the same basic mechanism (different wavelength photoreceptors using same rhodopsin chemistry)
- Quantitative variations along a continuum (multiple thermoreceptors at different temperature ranges)
- Fully integrated processing with no segregation
- Loss of any component eliminates entire function
- Designed as unified architecture despite apparent complexity

**Judgment Principle:** When uncertain, consider functional independence. If component A can be eliminated without affecting component B's operation, they are likely distinct subsystems. If A's loss degrades or eliminates B's function, they may be parts of a unified subsystem.

### 5.3 Control vs. Feedback Disambiguation

**Controls:**
- Established BEFORE operation begins
- Define what states/behaviors are permissible
- Constrain the space of possibility
- Embedded in system structure or design
- Example: PMNS mixing angles are controls; quantum axioms are controls

**Feedback:**
- Generated DURING/AFTER operation
- Report what states/behaviors occurred
- Provide information about actual outcomes
- Emerge from system operation
- Example: MSW matter potential is active feedback; stable particle existence is passive feedback

**Integration Example:** A thermostat contains BOTH:
- Control: The set point (70°F) defines target state BEFORE operation
- Feedback: The temperature sensor provides information about actual state DURING operation
- The control signal (turn heater on/off) results from COMPARING control parameter against feedback information

**Common Confusion:** Systems that embody both functions (like thermostats, quantum field theories) can make the distinction seem arbitrary. The key is temporal and functional orientation: controls look forward (what SHOULD happen), feedback looks backward (what DID happen).

### 5.4 Active vs. Passive Feedback Identification

**Active Feedback Indicators:**
- Explicit sensors or measurement systems
- Dedicated signaling pathways carrying feedback information
- Real-time or near-real-time response to deviations
- Proportional or threshold-based correction signals
- Can be disrupted without eliminating system operation (system operates worse but persists)
- Examples: Self-energy corrections, MSW matter effects, baroreceptor reflexes

**Passive Feedback Indicators:**
- No explicit signaling loop required
- System's mere continued existence constitutes the feedback
- Operates at long timescales (hours, days, years, millennia, cosmic time)
- Indicates that internal variety matches environmental variety per Ashby
- Cannot be disrupted without system failure (it IS the viability signal)
- Examples: Stable vacuum persistence, conservation law compliance, CPT invariance

**Both Modes Present:** All viable systems exhibit both. If you cannot identify passive feedback, you're not looking at long enough timescales or considering existential persistence. If you cannot identify active feedback, the system likely has rapid corrective mechanisms you haven't yet discovered (or is purely passive like a crystal structure).

### 5.5 Common Analytical Pitfalls

**Pitfall 1: Over-Fragmenting Elements**
- Identifying dozens of "subsystems" that are really minor variations
- Solution: Apply functional independence test; consolidate variants of unified mechanisms

**Pitfall 2: Conflating Elements**
- Treating feedback as control or vice versa
- Treating processing as input/output
- Confusing interface crossing with processing transformation
- Solution: Use temporal and directional tests (when does it occur? where does flow move?)

**Pitfall 3: Missing Recursive Structure**
- Stopping at surface-level element identification without exploring subsystem 7ES structure
- Solution: Select 2-3 key subsystems and perform full recursive analysis

**Pitfall 4: Ignoring Passive Feedback**
- Only counting explicit signaling loops as feedback
- Solution: Always ask "What confirms this system remains viable over long timescales?"

**Pitfall 5: Treating Environment as Static**
- Describing environment as single context rather than multiple overlapping domains
- Solution: Enumerate distinct environmental aspects providing different resources, constraints, or varieties

**Pitfall 6: Underestimating Interface Complexity**
- Missing boundary-crossing mechanisms, especially in quantum systems
- Solution: Look for basis transformations, regime transitions, measurement interfaces
