## Executive Summary

This update completes the domain-specific weighting system for Fundamental Design Principles (FDP) scoring. MRF v1.7 documented weight vectors for three system types (Economic, Technological, Ecological). This update adds five additional system types (Governance/Political, Social/Community, Financial, Infrastructure, Informational/Media) to provide complete domain coverage.

**What Changes:** Addition of five domain-specific weight vectors following the established structure (one FDP at 3.0, two at 2.0, five at 1.0).

**What Stays the Same:** All FDP definitions, individual FDP calculation formulas, classification thresholds, and integration with other framework components remain unchanged.

**Impact:** The methodology now provides systematic weight allocation across all eight system types, eliminating reliance on ad-hoc adjustments for domains previously without formal guidance. Systems audited using proper domain weights typically show stronger classification signals, as critical FDP failures receive appropriate emphasis.

**Backward Compatibility:** Full. Previously audited systems remain validly classified. The three domains documented in v1.7 (Economic, Technological, Ecological) are unchanged. The five newly documented domains formalize what was previously addressed through auditor judgment.

## Rationale for Update

### Incomplete Domain Coverage in v1.7

MRF v1.7 established domain-specific weighting for three system types but left five types without formalized guidance:

**Documented in v1.7:**
- Economic systems (RE=3.0, SP=2.0, CLM=2.0)
- Technological systems (ET=3.0, IH=2.0, AR=2.0)
- Ecological systems (CH=3.0, CLM=2.0, AR=2.0)

**Previously undocumented:**
- Governance/Political systems
- Social/Community systems
- Financial systems
- Infrastructure systems
- Informational/Media systems

This incomplete coverage meant that audits of governance systems, social organizations, financial institutions, infrastructure providers, and media platforms lacked the systematic weight allocation that economic, technological, and ecological systems received.

### Completing the Framework

This update extends the established weighting structure to the five remaining system types. Each domain now has a formal weight vector that emphasizes the FDPs most critical to that domain's core failure mode, maintaining structural consistency with the three domains documented in v1.7.

The weighting structure remains uniform across all eight domains:
- One FDP receives 3.0 weighting (primary failure indicator)
- Two FDPs receive 2.0 weighting (secondary failure indicators)
- Five FDPs receive 1.0 weighting (standard emphasis)

This consistency enables systematic methodology application across all system types while allowing domain-specific emphasis where empirically warranted.

## Methodology: Domain Classification

### Classification Criteria

The methodology classifies systems based on primary function and core failure mode. When a system's primary function determines which of the eight domains applies, the corresponding weight vector is used for FDP calculation.

**Primary Function Mapping:**

1. Creating/distributing economic value → Economic
2. Processing information/making algorithmic decisions → Technological
3. Managing natural resources/environmental impacts → Ecological
4. Governing/regulating human behavior → Governance/Political
5. Building community/facilitating collective action → Social/Community
6. Managing financial assets/risk/capital → Financial
7. Providing essential services/utilities → Infrastructure
8. Creating/distributing information/content → Informational/Media

### Multi-Domain Systems

When systems span multiple domains, classification prioritizes the failure mode that would cause the most severe systemic harm if manifested.

**Example:** A social media platform exhibits characteristics of:
- Technological domain (recommendation algorithms)
- Informational/Media domain (content distribution)
- Economic domain (advertising business model)

**Failure mode severity analysis:**
- Algorithmic bias (Technological) → Moderate harm
- Misinformation spread (Informational/Media) → Severe harm
- Advertiser exploitation (Economic) → Moderate harm

**Classification outcome:** Informational/Media (primary domain for weight vector application)

The methodology documents multi-domain characteristics while applying the weight vector from the domain whose failure mode presents the greatest risk.

## Methodology: Weighted Global FDP Calculation

### Formula

```
FDP_global = Σ(FDP_i × Weight_i) / Σ(Weight_i)
```

Where:
- FDP_i = individual FDP score (0-10 scale)
- Weight_i = domain-specific weight for that FDP
- Σ = sum across all 8 FDPs

### Calculation Sequence

The methodology follows this sequence for FDP assessment:

1. **Domain Classification** - System classified into one of eight domains based on primary function and core failure mode
2. **Weight Vector Selection** - Corresponding weight vector applied based on domain classification
3. **Individual FDP Scoring** - Each FDP scored 0-10 using standard FDP formulas
4. **Weighted Aggregation** - Individual scores multiplied by weights and summed
5. **Normalization** - Sum divided by total weight to produce Global FDP score
6. **Data Quality Assessment** - Quality penalty applied if average data quality below threshold
7. **Classification** - Global FDP compared to thresholds (Natural/Hybrid/Unnatural)

### Worked Example: SAVE Act (Governance/Political System)

**Step 1: Domain Classification**

System type: Legislation governing voter registration requirements
Primary function: Regulating human behavior (voting)
Core failure mode: Power concentration (federal override of state election systems)
**Domain:** Governance/Political

**Step 2: Weight Vector Selection**

Governance/Political weights: DA=3.0, RE=2.0, AR=2.0, all others=1.0

**Step 3: Individual FDP Scoring**

Based on detailed audit analysis:
- SP (Symbiotic Purpose) = 0.5
- AR (Adaptive Resilience) = 1.2
- RE (Reciprocal Ethics) = 0.8
- CLM (Closed-Loop Materiality) = 2.1
- DA (Distributed Agency) = 0.5
- CH (Contextual Harmony) = 3.0
- ET (Emergent Transparency) = 2.5
- IH (Intellectual Honesty) = 1.8

**Step 4: Weighted Aggregation**

| FDP | Raw Score | × Weight | = Weighted Score |
|-----|-----------|----------|------------------|
| SP | 0.5 | × 1.0 | = 0.5 |
| AR | 1.2 | × 2.0 | = 2.4 |
| RE | 0.8 | × 2.0 | = 1.6 |
| CLM | 2.1 | × 1.0 | = 2.1 |
| DA | 0.5 | × 3.0 | = 1.5 |
| CH | 3.0 | × 1.0 | = 3.0 |
| ET | 2.5 | × 1.0 | = 2.5 |
| IH | 1.8 | × 1.0 | = 1.8 |

Weighted sum = 15.4

**Step 5: Normalization**

Total weights = 1.0 + 2.0 + 2.0 + 1.0 + 3.0 + 1.0 + 1.0 + 1.0 = 12.0

FDP_global = 15.4 / 12.0 = 1.28

**Step 6: Data Quality Assessment**

Average data quality = 4.2 (above 3.0 threshold, no penalty applied)

**Step 7: Classification**

FDP_global = 1.28 falls in range 0.0-4.9 → Unnatural (Collapse-Prone)

**Significance:** The catastrophically low DA score (0.5) receives triple emphasis (×3.0) under Governance/Political weighting, properly reflecting power concentration as the primary failure mode for legislation.

## Data Quality Integration

### Quality Assessment Criteria

The methodology incorporates data quality assessment to prevent false precision when underlying evidence is weak:

| Quality Factor | Scale | Criteria |
|----------------|-------|----------|
| Data Completeness | 1-5 | 5=Complete documentation, 1=Major evidence gaps |
| Source Reliability | 1-5 | 5=Primary sources/direct measurement, 1=Unverified claims |
| Measurement Accuracy | 1-5 | 5=Precise quantitative metrics, 1=Rough estimates |
| Time Relevance | 1-5 | 5=Current/recent data, 1=Outdated information |

### Quality Penalty Application

When average quality score falls below 3.0, the methodology applies a 0.5 point penalty to Global FDP.

**Example:**
- Calculated FDP_global = 1.28
- Average data quality = 2.8 (below threshold)
- Adjusted FDP_global = 1.28 - 0.5 = 0.78

### Precision Reporting

The methodology matches reported precision to data quality:

- High quality (average ≥ 4.0): Report to two decimals (FDP_global = 1.28)
- Moderate quality (3.0-3.9): Round to one decimal (FDP_global = 1.3)
- Low quality (<3.0): Report with uncertainty notation (FDP_global ≈ 1.3 or 1.3 ± 0.5)

This prevents presentation of high-precision scores when underlying data quality cannot support such precision.

## Impact on Existing Audits

### Backward Compatibility

All systems audited under MRF v1.7 remain validly classified. The three domains documented in v1.7 (Economic, Technological, Ecological) are unchanged in v1.8. The five newly documented domains formalize what was previously addressed through judgment-based weight allocation.

### Example: SAVE Act Audit Conducted Under v1.7

**Original approach:**
- Governance system audited without formalized domain weights
- Auditor used judgment-based allocation: SP=3.0, RE=3.0, IH=2.0, ET=2.0, DA=2.0
- Reasoning: Emphasized purpose, ethics, honesty based on auditor assessment
- Result: Low Global FDP → Unnatural classification

**v1.7 methodology:**
- Governance/Political domain weights: DA=3.0, RE=2.0, AR=2.0, others=1.0
- Systematic emphasis on power distribution, fairness, self-correction
- Calculation differences:
  - DA emphasis increased (2.0 → 3.0): Catastrophic DA=0.5 score receives greater weight
  - AR emphasis increased (1.0 → 2.0): Poor AR=1.2 score receives greater weight  
  - SP, IH, ET emphasis reduced: Overemphasis corrected

**Recalculation result:** FDP_global = 1.28 (lower than judgment-based approach)

**Classification:** Remains Unnatural, but magnitude of dysfunction now more accurately reflected through systematic weighting

**Audit validity:** The original audit correctly identified the system as Unnatural/Collapse-Prone. The v1.8 methodology strengthens rather than contradicts this conclusion by providing systematic rather than judgment-based weight allocation.

## Validation Notes

### Empirical Grounding

Weight allocations reflect observed failure patterns across system types documented in framework case studies. Each domain's heavily weighted FDPs (3.0 and 2.0) correspond to the failure modes most commonly observed when that system type exhibits collapse-prone characteristics.

**Example:** Governance systems that concentrate power (low DA) and lack self-correction (low AR) while distributing burdens unfairly (low RE) consistently exhibit the collapse patterns the framework identifies as Unnatural. The Governance/Political weight vector (DA=3.0, RE=2.0, AR=2.0) emphasizes these empirically-validated failure indicators.

### Structural Consistency

All eight domains follow the same weighting structure (one FDP at 3.0, two at 2.0, five at 1.0) to maintain methodological consistency. This uniform structure prevents arbitrary weight inflation and ensures comparability across domains while allowing appropriate emphasis on domain-specific failure modes.

### Future Refinement

The weight vectors documented here represent the current state of framework understanding based on available case studies and empirical validation. Future updates may adjust weights if accumulating evidence demonstrates different FDPs provide stronger failure prediction for specific domains. Any such refinements will be documented with empirical justification and maintained through version control.
